# =============================================================================
# SECTION 3 — bcrypt
# =============================================================================
#
# bcrypt works differently from MD5 and SHA-256 in two important ways:
#
#   1. It is non-deterministic: the same password hashed twice produces two
#      different hashes. You can NEVER re-generate the stored hash — you can
#      only check whether a candidate matches it.
#
#   2. It has a cost factor (the number after $2b$). Higher cost = more
#      computation per check. rounds=12 is the production standard;
#      rounds=4 is fast enough for testing.
#
# Install bcrypt first:  pip install bcrypt
#
# How to verify in Python:
#
#   import bcrypt
#   stored_hash = b"$2b$04$0/TlUcBmJUrSOZZJRNjhEOO/x56sM.IzATPxeHsqEv8DnRrD2tx5i"
#   bcrypt.checkpw(b"hello", stored_hash)
#   # should return: True
#
#   bcrypt.checkpw(b"wrong", stored_hash)
#   # should return: False
#
# Note: pass bytes (b"hello"), not a string ("hello"), to checkpw.
#
# Format: password  →  bcrypt_hash  [cost rounds]
# -----------------------------------------------------------------------------

hello     →  $2b$04$0/TlUcBmJUrSOZZJRNjhEOO/x56sM.IzATPxeHsqEv8DnRrD2tx5i   [rounds=4,  fast]
password  →  $2b$12$uY7GQ5uojk9PmgSaJkoYVOCuUzDjV1aCkvr/fEu5LuXjV6YVCGfCm   [rounds=12, slow]

# SECTION 5 — Character set reference
# =============================================================================
#
# Use these numbers when calculating search spaces and estimating crack times.
#
# Character set              Size   Characters
# ─────────────────────────────────────────────────────────────────────────────
# Digits only                  10   0123456789
# Lowercase only               26   abcdefghijklmnopqrstuvwxyz
# Uppercase only               26   ABCDEFGHIJKLMNOPQRSTUVWXYZ
# Digits + lowercase           36   0-9, a-z
# Digits + lower + upper       62   0-9, a-z, A-Z
# Full printable ASCII         95   all printable characters (space to ~)
#
# Search space formula:  charset_size ^ password_length
#
# Examples:
#   4-digit PIN         : 10^4  =              10,000
#   5-digit PIN         : 10^5  =             100,000
#   6-digit PIN         : 10^6  =           1,000,000
#   4 lowercase letters : 26^4  =             456,976
#   5 lowercase letters : 26^5  =          11,881,376
#   6 lowercase letters : 26^6  =         308,915,776
#   8 lowercase letters : 26^8  =     208,827,064,576
#   8 chars, full set   : 95^8  =   6,634,204,312,890,625
#
# Python speed reference (approximate, varies by machine):
#   Your brute forcer will run at roughly 500–2,000 ZIP attempts per second.
#   At 1,000 attempts/second:
#     10,000 attempts  →  ~10 seconds
#     100,000 attempts →  ~100 seconds (1.7 minutes)
#     1,000,000 attempts → ~1,000 seconds (17 minutes)
#     11,000,000 attempts → ~3 hours
#
#   This is intentionally slow. It makes the difference between attack
#   strategies visible in your timing data. A GPU-based tool like Hashcat
#   runs billions of attempts per second — but you are not allowed to use it.
